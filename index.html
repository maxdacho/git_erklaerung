commit

Ein Commit ist ein Schnappschuss der aktuellen Version eines Projektes.

branch

Branch ist ein Zweig auf dem Commits liegen.
Ein „Zeiger“ markiert den letzten/aktuellen Commit.
Der Standard Branch heißt "master“ oder "main".

checkout

Git checkout ermöglicht es zwischen branches zu wechseln.

cherry-pick

Bei Cherry-Picking wird ein Commit aus einem Branch ausgewählt und auf einen anderen angewendet.
Kann hilfreich sein um Änderungen Rückgängig zu machen.

reset

Git reset setzt den Commit-Verlauf zurück.

revert

Git revert macht ebenfalls Änderungen rückgängig, durch Ermittlung, wie Änderungen umgekehrt werden können.

rebase

Git Rebase kann ebenfalls Veränderungen auf eine anderen Branch übertragen, umschreibt jedoch den Verlauf.
Ist der komplexere Befehl im Vergleich zu git Merge.
Es lässt sich die Struktur des Reposetories organisieren.
Dies ist nützlich um gegen Projektende unnötige Daten zu löschen.
Durch die Verwendung von ~ und ^ kann man direkt zu bestimmten Vorgängern Commits springen.
Dies funktioniert durch Angabe einer Nummer zur welcher Version man genau möchte.

merge

Mit git Merge lassen sich zwei Entwicklungszweige zu einem zusammenführen.
Aktuelle Branch wird aktualisiert.
Veraltete Ziel-Branch bleibt unverändert und kann gelöscht werden.
