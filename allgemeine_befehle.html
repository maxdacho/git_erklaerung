Allgemeine Befehle

init

Mit Git init kann ein neues Repository angelegt werden.
Der Befehl kann außerdem noch zur Konvertierung eines bestehenden, nicht versionierten Projekts oder zur Initialisierung eines neuen, leeren Repos genutzt verwendet werden.

config

Mit Git config kann ein Konfigurationsnamens aufgerufen werden, um den definierten Wert für diesen Namen anzuzeigen.

commit

Ein Commit ist ein Schnappschuss der aktuellen Version eines Projektes.

branch

Branch ist ein Zweig auf dem Commits liegen.
Ein Zeiger markiert den letzten/aktuellen Commit.
Der Standard Branch heißt master oder main.

checkout

Git checkout ermölglicht es zwischen branches zu wechseln.

cherry-pick

Bei Cherry-Picking wird ein Commit aus einem Branch ausgewählt und auf einen anderen angewendet.
Kann hilfreich sein um Änderungen Rülckgängig zu machen.

reset

Git reset setzt den Commit-Verlauf zurülck.

revert

Git revert kann Änderungen von bereits existierenden Commits rückgangig machen.
Hierbei werden jedoch nicht die Daten gelöscht.
Git stellt lediglich Änderungen mit dem gegenteiligen Effekt an und macht dabei den angegebenen Commit rückgängig.

rebase

Git Rebase kann ebenfalls Veränderungen auf eine anderen Branch übertragen, umschreibt jedoch den Verlauf.
Ist der komplexere Befehl im Vergleich zu git Merge.
Es lässt sich die Struktur des Repositories organisieren.
Dies ist nützlich um gegen Projektende unnötige Daten zu löschen.
Durch die Verwendung von ~ und ^ kann man direkt zu bestimmten Vorgängern Commits springen.
Dies funktioniert durch Angabe einer Nummer zur welcher Version man genau möchte.

push

Mit Git push werden lokale Änderungen in einem zentralen Repository veröffentlicht und hochgeladen.
Dadurch können die neuesten Änderungen am lokalen Repository für die Teammitglieder erreichbar gemacht.

clone

Durch Git clone können bereits existierende Repositories auf den eigenen lokalen Computer gedownloaded werden.

fetch

Mit dem Befehl Git fetch wird lediglich neue Daten von einem remote Repository gedownloaded.
Es integriert jedoch nicht die neuen Daten in die working files.
Der Vorteil dabei ist, dass durch git fetch niemals etwas umgeordnet beziehungsweise verstellt wird.

pull

Der Befehl Git pull wird benutzt um den aktuellen HEAD branch mit den neusten Änderungen vom remote Server geupdated wird.
Durch git pull werden demnach neue Daten nicht nur gedownloaded, sondern auch direkt in die aktuellen working copy files integriert.

merge

Mit git Merge lassen sich zwei Entwicklungszweige zu einem zusammenführen.
Aktuelle Branch wird aktualisiert.
Veraltete Ziel-Branch bleibt unverändert und kann gelöscht werden.

add

Der git add fügteine Änderung aus dem Arbeitsverzeichnis zur Staging-Umgebung hinzu.
Die Änderungen werden jedoch erst aufgezeichnet, wenn der Befehl git commit ausgeführt wird.
